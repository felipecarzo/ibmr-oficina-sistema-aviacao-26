# 2026-05-06 — Sessão: Beat-Sync Mode

## Contexto
Sessão iniciada com objetivo de implementar beat-sync auto-cut inspirado no Pulse Edit Pro.
Análise confirmou que o Pulse Edit Pro provavelmente usa DSP (não AI real) — marketing.

## O que foi feito

### Análise do projeto existente
- Lido todo o código em `src/` e `main.py`
- Confirmado que `audio_analyzer.py` já usa `librosa.beat_track()` mas descarta os beats
- Beats eram calculados só para detectar "Party Mode" — nunca viravam cortes

### Decisões de arquitetura
- Beat-sync como **modo separado** (flag `--music`), não integrado ao pipeline AI
- Motivo: pipeline AI leva ~30min; beat-sync leva ~10s — workflows diferentes
- Output: CSV de markers (não EDL) — usuário encaixa manualmente no DaVinci
- Formato CSV compatível com importação nativa do DaVinci Resolve

### Implementado: `src/beat_sync.py`
- `detect_beats()` — `librosa.beat_track` + `onset_detect` com backtrack=True
- `scan_takes()` — lista vídeos na pasta fornecida
- `export_beat_markers()` — gera CSV com timecodes SMPTE
- `run_beat_sync()` — orquestra tudo, aceita `use_onsets=True` para ritmo mais frenético

### Atualizado: `main.py`
- `--input` virou opcional (não quebra modo AI)
- Novos flags: `--music`, `--takes`, `--fps`, `--onsets`
- Early return no modo beat-sync (não carrega modelos AI pesados)
